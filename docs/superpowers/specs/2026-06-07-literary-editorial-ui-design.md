# Reader Web 文学编辑室风格前端美化设计规格

日期：2026-06-07

## 目标

将 Reader Web 从基础工具界面升级为“文学编辑室”风格的本地阅读器。界面应安静、克制、纸面感强，适合长时间阅读；书架像私人藏书台，阅读页像排版良好的书页，设置与统计面板像专业阅读札记和排版控制台。

本次只调整前端视觉和必要的结构类名，不改变书籍解析、存储、阅读进度、导入导出协议等核心业务逻辑。

## 设计方向

- 视觉关键词：纸面、墨色、深绿、铜色、细线、低饱和阴影、书脊、章节索引。
- 主背景使用温润纸色，不使用纯白背景。
- 强调色从通用蓝色改为深绿，少量铜色用于收藏、封面书脊和统计强调。
- UI 元素保留功能密度，但降低工具栏和弹窗的“后台感”。
- 动效克制，主要用于页面进入、书卡 hover、抽屉和弹层进入。

## 设计系统

修改 `src/styles/variables.css`：

- 增加纸面色、墨色、深绿、铜色、暗红等语义变量。
- 保留现有变量名，避免大范围 JSX 修改。
- 重设 `--accent-color` 为深绿。
- 增加通用控制变量：
  - `--surface-paper`
  - `--surface-raised`
  - `--surface-muted`
  - `--text-muted`
  - `--accent-secondary`
  - `--danger-color`
  - `--focus-ring`
  - `--paper-grain`
- 白天、夜间、护眼主题都应有对应背景、正文、边框和强调色。

修改 `src/index.css`：

- 设置全局纸面背景和更适合阅读器的字体栈。
- 为按钮、输入框、选择框提供统一焦点态。
- 增加轻量纸纹背景，不引入图片资源。
- 保持 `html/body/#root` 当前高度和滚动约束，避免影响阅读页布局。

## 书架页

修改 `src/pages/BookShelf.jsx` 与 `src/pages/BookShelf.css`。

### 结构与信息层级

- 页眉增加副标题，说明这是本地藏书空间。
- 页眉增加简单统计：总藏书数、收藏数、已开始阅读数。
- 主操作为“导入书籍”，其他配置和统计按钮为次级操作。
- 搜索、排序、视图切换、选择模式整合为“书目工具栏”视觉。

### 书卡

- 网格卡片使用更稳定的书籍比例。
- 缺失封面时使用书脊式占位封面：深绿/铜色渐变、首字、细线装饰。
- 进度条作为书签线处理，降低厚度和视觉噪音。
- 收藏按钮为右上角小角标，active 使用铜色。
- 删除按钮默认弱化，保留清晰危险色。

### 列表模式

- 列表项像书目卡片，封面缩略图保留书脊质感。
- 标题、作者、进度、操作在同一行内保持清楚层级。
- 移动端列表操作允许换行，不挤压标题。

### 空状态

- 空书架改为纸面导入卡片。
- 文案保留简洁，主按钮明确指向导入第一本书。

## 导入页

修改 `src/pages/Import.jsx` 与 `src/pages/Import.css`。

- 页面呈现为编辑桌上的稿纸/书页。
- 拖拽区使用纸面背景、虚线边框、内阴影和深绿 hover/drag 状态。
- 替换 emoji 图标为 CSS 书本符号，避免平台差异。
- loading 文案改为“正在拆封书页...”
- 错误态为批注条样式。

## 阅读页

修改 `src/pages/Reader.jsx`、`src/pages/Reader.css` 以及必要的 reader 子组件。

### 页面壳

- 顶部栏改为低对比纸面工具栏。
- “返回”“禅”“设置”保持原有功能，视觉上使用描边/轻按钮。
- 阅读主区域增强纸面感，但不包成厚重卡片。

### 正文

- TXT 正文维持现有滚动和进度逻辑。
- 正文容器优化 padding、最大宽度、行高和文字抗锯齿。
- 白天主题使用暖纸色；夜间主题使用深墨色；护眼主题使用旧书页色。

### 底部栏

- 改成阅读状态条：目录、进度、上一章/页、下一章/页。
- disabled 状态必须清楚，不能只靠透明度。
- 移动端避免按钮文本挤压进度。

### 目录抽屉

- 目录面板呈现为章节索引。
- 当前章节使用左侧深绿细线和淡底色。
- 卷标题更像分组标题，章节列表更轻。

### 禅模式

- 保留最弱退出按钮，不新增干扰元素。
- 边界提示卡片改为纸面提示，不破坏沉浸感。

## 设置面板

修改 `src/components/style-panel/StylePanel.css`。

- 先修复移动端 `@media` 结构，避免自定义字体样式只在小屏生效。
- 面板宽度扩大到约 420px，移动端使用底部抽屉式布局。
- Tab 改为分段控件。
- 主题预览像小书页，不是普通色块。
- 字号、字重、行高、宽度按钮使用稳定尺寸和清晰 active 态。
- 颜色预设保留圆形色样，但边框和 focus 态改为编辑室风格。

## 统计面板

修改 `src/components/StatsPanel.jsx` 与 `src/components/StatsPanel.css`。

- 标题改为“阅读札记”。
- 统计卡片像手账记录，数字使用更强的衬线表现。
- 热力图颜色从纸色逐步过渡到深绿，避免荧光感。
- 弹层使用纸面背景、细边框和柔和阴影。

## 动效

- 使用 CSS 动效，不引入新依赖。
- 书卡进入：轻微淡入上移。
- 书卡 hover：轻微上移、封面阴影增强。
- 拖拽区 drag-over：边框和背景变深。
- 弹层和抽屉：淡入或轻微位移动画。
- 尊重 `prefers-reduced-motion`，在减少动效偏好下禁用非必要动画。

## 响应式

- 书架页在 720px 以下页眉纵向排列。
- 工具栏在窄屏拆为搜索一行、操作一行。
- 书卡网格最小宽度约 150px。
- 阅读页底部栏在窄屏中允许换行或压缩间距，进度不遮挡按钮。
- 设置面板和统计面板在窄屏使用接近全屏/底部抽屉布局。

## 实施分组与提交

1. `docs`: 写入本设计规格文档。
2. `style(theme)`: 实现全局变量、基础控件、纸面背景和焦点态。
3. `style(shelf-import)`: 美化书架页和导入页。
4. `style(reader-panels)`: 美化阅读页、目录、设置面板和统计面板。
5. `chore`: 运行 lint、test、build，如验证中需要小修复则提交修复。

每完成一个分组都创建一次 git commit。

## 验收标准

- `npm run lint` 通过。
- `npm run test` 通过。
- `npm run build` 通过。
- 书架页、导入页、阅读页、设置面板、统计面板视觉统一。
- 白天、夜间、护眼主题均可读，正文对比度不低。
- 移动端不存在明显按钮挤压、文字溢出或弹层遮挡。
- 不改变解析、存储、进度和导入导出核心行为。
